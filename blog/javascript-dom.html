My Non-technical friend:
Explain control flow and loops for me!

Consider the process of going out to buy groceries. Imagine you first create a list of the items you need. Then, you drive to the supermarket, grab a trolley, and proceed to pick up each item one by one. If the first item is available, you pick it, otherwise, you may look for an alternative brand. Finally, if they've run out of that product, you skip that item and move on to the next one. You won’t wait a week until they restock it. At the end, you head to the cashier to pay for all the items. (Loops)
Notice how each step in this process must be completed in a specific order.
Can you pay for the items before selecting them? No!
Can you select items before even arriving at the supermarket? No!
Every action is dependent on the completion of the previous step.
This step-by-step process reflects how actions in programming are carried out in a set order, following certain conditions or rules.

My Non-technical friend:
Describe what the DOM is and an example of how you might interact with it.

Let's revisit the house we built in the first question and find something you bought in the second one!

This house includes a kitchen. Within the kitchen, there's a pantry with shelves, each stocked with various groceries and items.

Now, if I wanted to find the cocoa powder, you would advise me to go to the kitchen, open the pantry, and locate it on the second shelf. This analogy mirrors the Document Object Model (DOM).

In the context of a web page, the DOM serves a similar purpose. If you wish to alter text within a paragraph or assign an action to a button click, you would utilize the DOM to locate these elements (such as the paragraph or button) and then implement your desired changes or actions.

Thus, the DOM essentially functions as a tool for computer programs, like JavaScript, to comprehend and interact with the elements on a web page.

This is the DOM Hierarchy

In order to modify elements within an HTML document, we utilize the Document Object Model (DOM), which serves as a programming interface for web content. Rather than requiring a 'direction', the DOM provides methods to precisely target and manipulate specific elements. This can be done by selecting elements based on various attributes such as tag names, IDs, or classes. For instance, if we have a <div> element with an ID attribute like this: <div id="myDiv">Hello, world!</div>, we can employ JavaScript to select this element by its ID and alter its content. The method document.getElementById("myDiv") selects the element with the ID 'myDiv', allowing us to subsequently change its properties or content as needed.

Explain the difference between accessing data from arrays and objects.

Are you still in the kitchen?

Great! Arrays are like shelves in the pantry; you can find items based on their position. For example, on the second shelf from the left, you have sugar, cocoa powder, flour, and shredded coconut! So if you ask me to hand you the second item, it’s always the cocoa powder!

Objects are like labeled containers in the pantry. You have a drawer labeled "spices" in your pantry. If you open it, you can find many spices like chili flakes, oregano, thyme, or dill. But how do you find which one is oregano? By reading its label!!

So, with arrays, you locate items by their position, while with objects, you find them by their labels.

My Non-technical friend:
Explain what functions are and why they are helpful.

In programming, a function is a reusable block of code that performs a specific task. It takes input(s) as parameters, performs some operations on them and returns a result. Functions are used to organize code, make it more readable, and avoid repetitive tasks.
